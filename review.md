## ⚠️ Серьёзные проблемы

### 2. Несоответствие цветовой схемы
- **Требование (design-system.md):** Primary color `#1A1A1A` (чёрный)
- **Реальность (style.css):** Primary color `#007aff` (синий iOS)
- **Где:** `:root { --color-primary: #007aff; }`
- **Исправление:** Заменить на `--color-primary: #1A1A1A;`

### 3. Не все типы голосований реализованы в UI
- **Требование (research.md):** 
  - Обычное голосование (за/против)
  - Тайное голосование
  - Смена администратора (50%+1, 72 часа)
  - Удаление группы (50%+1)
- **Проблема:** В create-voting-modal есть только select, но нет специальных UI для:
  - Голосования за добавление участника вопреки админу
  - Голосования за удаление участника
- **Исправление:** Добавить отдельные flow/экраны для специальных типов голосований

### 4. Отсутствует переключатель языка
- **Требование (research.md):** Языки: EN / UK
- **Реальность:** Только украинский язык, переключателя нет
- **Где:** Добавить в экран профиля или настройки

### 5. Нет Google Sheets экспорта
- **Требование (research.md):** Google Sheets экспорт истории голосований
- **Реальность:** Функционал не реализован
- **Где:** Добавить кнопку "Експорт в Google Sheets" в экран завершённых голосований

## 📝 Средние проблемы

### 6. Неполная реализация статусов голосований
- **Требование (research.md):** Цветовая маркировка: 🟡 безрезультатное, 🟢 принято, 🔴 отклонено
- **Реальность:** В CSS есть `.voting-status.pending`, `.accepted`, `.rejected`, но:
  - Нет логики определения "безрезультатного" (равное количество за/против)
  - Нет отображения процентного порога принятия (50%+1)

### 7. ~~Отсутствует механизм 6-значного ID группы~~ ✅ ЧАСТИЧНО ИСПРАВЛЕНО
- **Требование (research.md):** 6-значный ID группы
- **Реальность:** 
  - ✅ Генерация ID при создании группы: `Math.random().toString().slice(2, 8)`
  - ✅ Валидация 6 цифр при вводе: `groupId.length !== 6`
  - ⚠️ Нет проверки что ID уникален (только mock-данные)

### 8. Неполная форма профиля
- **Требование (research.md):** Профиль: имя, фамилия, адрес объекта, номер дома/квартиры, телефон
- **Реальность:** В profile-setup-screen поля есть, но в profile-screen отображаются не все данные:
  - Нет разделения на "адрес объекта" и "номер квартиры" — склеено в одно поле
  - Нет отображения email в структурированном виде

### 9. ~~Отсутствует логика запросов на вступление~~ ✅ ИСПРАВЛЕНО
- **Требование (research.md):** Запрос на вступление → одобрение админом
- **Реальность:** 
  - ✅ `approveRequest()` и `rejectRequest()` реализованы
  - ✅ Индикация запросов для админа в group-detail-screen
  - ✅ Уведомление о запросе создаётся при `joinGroup()`

## 💡 Улучшения (Nice to have)

### 10. UX/UI улучшения
- **Бейдж "Тайное голосование":** Нужен визуальный индикатор на карточке голосования
- **Обратный отсчёт:** Для активных голосований показывать "Осталось: 2 дня 4 часа"
- **Пустые состояния:** Нет экранов для "Нет активных голосований", "Вы не состоите в группах"
- **Подтверждение выхода:** Нет модалки подтверждения при выходе из группы/аккаунта

### 11. Доступность (a11y)
- Нет `aria-label` для иконок-кнопок
- Нет `role` атрибутов для семантики
- Нет фокус-стилей для клавиатурной навигации

### 12. Адаптивность
- На desktop (>480px) контейнер ограничен 480px — ✅ хорошо
- Но нет 2-колоночного режима для планшетов (как указано в design-system.md)
